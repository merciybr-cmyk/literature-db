# 작가별 탭 + 저작권 정보 설계 (2026-06-23)

작품 DB에 등장하는 작가를 1급 개체로 다루는 전용 화면. 작가별로 ① 수록 작품 전체와 ② 저작권 관리 상태(만료/신탁/별도 요청 등)를 한 화면에서 본다. 저작권 자료는 외부 파일에서 동기화한다.

## 목적·배경

- 어떤 작가를 봤을 때 "저작권 처리를 누구와 해야 하나, 아니면 만료돼 자유 이용 가능한가"를 즉시 알 수 있어야 한다.
- 저작권 자료(`D:\교과서 문학 단행본 시리즈\저작권` 폴더)는 형식이 제각각이고 주기적으로 업데이트된다 → `works.json`처럼 별도 동기화 스크립트로 정규화한다.

## 데이터 소스

폴더 `D:\교과서 문학 단행본 시리즈\저작권`:

| 파일 | 시트/범위 | 의미 | 사용 항목 |
|---|---|---|---|
| `저작권 만료 저자_*.xlsx` | 저작권만료 | 만료 = 자유 이용(퍼블릭 도메인) | 저자명, 생몰연도, 사망연도, 만료시점 |
| `KOLAA 회원명단*.xlsx` | 신탁자목록 / 이미지 등록 | 한국문학예술저작권협회 신탁·관리 | 저작자명, 저작권자명, 확인, 비고, 추가사항(계약종료일) |
| `사이에이전시_*.xlsx` | 회원명단 | 사이저작권에이전시 관리 | 작가명(저작권자명), 가입시기. **저작물목록(42만행)은 미사용** |
| `남북저작권센터_*.xlsx` | 남북저작권센터 저작물목록 | 월북·북한 작가 등 관리 | 저작자명, 저작물 구분 |
| `별도사용료_요청작가.csv` *(신규 유지보수 파일)* | — | KOLAA 관리지만 **별도 사용료 요청** 작가 | 작가명, 갈래(산문/운문), 대표저작물, 적용시점 |

### PDF 소스 처리

`한국문학예술저작권협회_저작권 사용료 기준.pdf`(p3~5, "[별지] 사용료 별도 요청 기준")는 **폰트에 유니코드 매핑이 없어 텍스트 자동 추출이 불가능**하다. 따라서 자동 파싱하지 않고, 그 내용을 **구조화된 유지보수용 파일 `별도사용료_요청작가.csv`로 한 번 옮겨** 동기화 소스로 삼는다.

- 담는 범위: **플래그(별도 사용료 요청) + 갈래(산문/운문) + 대표저작물 + 적용시점**. 정확한 요율 표(금액)는 담지 않고, 앱에서는 "KOLAA 관리·별도 요율 요청 작가"로 표시 + PDF 참조 안내.
- 초기 데이터는 PDF 이미지(약 120~140명)에서 옮겨 채우되, **작가명 정확성은 사용자 검증 필수**(금액·법적 맥락). PDF 갱신 시 사용자가 이 CSV를 수정 → 재동기화.

## 컴포넌트

### 1. 동기화 파이프라인 — `scripts/sync-copyright.mjs`

- `npm run sync-copyright` 추가(`package.json`).
- 폴더의 xlsx 4종 + `별도사용료_요청작가.csv`를 읽어 **작가명으로 정규화 병합** → `public/copyright.json` 생성.
- 파일명은 날짜가 바뀔 수 있으므로 **glob/접두어 매칭**(`저작권 만료 저자*`, `KOLAA 회원명단*`, `사이에이전시*`, `남북저작권센터*`)으로 찾는다.
- `xlsx`(기존 의존성) 사용. ESM에서는 `XLSX.read(readFileSync(...), {type:'buffer'})` 형태.
- 출력 형태(작가명 키):

```json
{
  "김소월": { "matched": ["expired"], "expired": { "생몰연도": "1902~1934", "사망연도": 1934, "만료시점": "2005.01.01" }, "registries": [] },
  "백석":   { "matched": ["남북저작권센터"], "registries": [{ "source": "남북저작권센터", "저작물구분": "운문(시)" }] },
  "임철우": { "matched": ["KOLAA", "별도요청"], "registries": [{ "source": "KOLAA", ... }], "별도요청": { "갈래": "산문", "대표저작물": "...", "적용시점": "..." } }
}
```

### 2. 매칭·상태 유틸 — `src/utils/copyright.js`

- `normalizeAuthor(name)`: 공백 제거 등 정규화 키 생성.
- `lookupCopyright(authorBase, copyrightMap)`: 작품 DB의 `_authorBase` ↔ 자료 작가명을 **공백 제거 후 정확 일치**로 매칭. 미매칭은 `미확인`. (퍼지 매칭은 동명이인 오매칭 위험이 커서 배제. 화면에 "이름 기반 best-effort" 명시.)
- `copyrightStatus(record)`: 표시용 상태 배열 산출. 우선 강조 규칙:
  - 🟢 **만료(자유 이용)** — 가장 결정적, 별도 강조
  - 🔵 KOLAA · 🟣 사이에이전시 · 🟠 남북저작권센터 — 신탁·관리(처리 경로)
  - 🔴 **별도 사용료 요청**(KOLAA 위에 덧붙는 태그)
  - ⚪ **미확인** — 어느 자료에도 없음
- 한 작가가 여러 자료에 걸칠 수 있으므로 매칭된 소스를 **모두** 배지로 표시.
- Vitest 단위 테스트: 정규화, 매칭, 상태 판정, 다중 소스 케이스.

### 3. 화면 — `src/pages/AuthorsPage.jsx`

- **작가 디렉터리**(좌측/상단): 작품 DB에 등장하는 모든 작가(외국 작가 포함 — 대부분 `미확인`). 초성/가나다 + 수록 편수 순 정렬, 검색창(기존 `chosung.js` 재사용), 저작권 상태 필터 칩.
- **작가 상세**(선택 시):
  - ① 저작권 상태 배지 + 상세(만료시점 / 계약종료일 / 저작권자명 / 별도요청 갈래·대표저작물·적용시점 등). 별도요청 작가는 PDF 참조 안내.
  - ② 그 작가의 수록 작품 전체(작품명·교육과정·교과서). 작품 클릭 시 기존 `WorkDetailModal` 재사용.
- `NavBar`에 `['authors','작가별']` 추가, `App.jsx`에 라우팅 1줄(기존 탭 패턴).

### 4. 데이터 로딩

- `useWorks.js`와 동일 패턴으로 `public/copyright.json`을 로드(신규 `useCopyright.js` 또는 `useWorks` 확장). 배포/로컬 모두 정적 JSON 사용.

## 데이터 흐름

```
저작권 폴더(xlsx 4종 + 별도사용료 CSV)
   └─ npm run sync-copyright ─→ public/copyright.json
                                     │
앱 로드: useWorks(works) + useCopyright(copyrightMap)
   └─ AuthorsPage: works를 _authorBase로 그룹핑
        └─ 각 작가 = lookupCopyright(_authorBase, copyrightMap)
             ├─ 디렉터리: 이름·편수·상태 배지
             └─ 상세: 저작권 상세 + 수록 작품 목록(→ WorkDetailModal)
```

## 공통 원칙·회귀

- 교육과정 목록·정렬·장르는 `src/constants.js` 재사용. 작가 그룹핑 키는 기존과 동일한 `_authorBase`.
- 기존 4개 탭(검색·중복·비교·통계) 동작 회귀 없음. `copyright.json` 미존재 시 전 작가 `미확인`으로 안전 동작(앱이 깨지지 않음).
- 새 유틸 `copyright.js`에 Vitest 테스트 추가.
- 저작권 데이터는 금액·법적 맥락 → 작가명 매칭/별도요청 명단의 정확성은 사용자 검증 전제, 화면에 best-effort 고지.

## 범위 밖 (YAGNI)

- 정확한 사용료 요율 표(금액) 데이터화.
- 사이에이전시 저작물목록 42만 행 작품 단위 연결.
- 작가명 퍼지 매칭/별칭 사전(필요 시 후속).
- 작가 이름 클릭 → 상세 진입(🅑 진입점)은 이번 범위 밖, 후속으로 쉽게 추가 가능.
